Tips for better answers
✓ Always say the language and framework
✓ Paste the exact error message, not a summary
✓ Say "do not use external libraries" if you want pure code
✓ Add "explain each change" to understand what was fixed
✓ Say "production ready" for error handling + edge cases
✓ Use "step by step" for complex problems
✓ If output is cut off, just say "continue"
